Automation and Process

Typical Workflow for Code

  • Version Control - track changes
  • Feature Branches - isolate changes
  • Build Server - automatically test and deploy

We would probably never...

  • Make changes "live" on the server
  • Not keep our code in version control
  • Use a manual process for deployment

But what about...

  • Database?
  • Map Services?
  • Portal users/groups?
  • Server Provisioning?

We're only as strong as our weakest link!

How long would it take to bring your project back up from scratch?

  • In 2016, the norm is minutes or seconds!

Tools

  • SQL scripts, Liquibase, Flyaway, SSDT
  • ArcPy.Mapping, ArcObjects
  • Portal API
  • Vagrant, Packer, Docker

Think scripting first!

What does this change?

  • Local database and AGS VMs on your dev laptop (iteration time)
  • Fewer bottlenecks with understaffed IT
  • Helps understand data dependencies
  • What about our clients?
