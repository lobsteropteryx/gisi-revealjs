The Value Proposition of Architecture

What is architecture?

  • Code structure and interaction
  • Infrastructure
  • Testing, Deployment

Good architecture lets us:

  • Write fewer bugs
  • Respond to changes (team, business, priority)
  • Continue to add features quickly as the project grows

It's about time!

Velocity

* This is a hard metric to get at!

Have you ever?

  • Found a bug in production?
  • Seen your estimates for new features go up as a project grew?
  • Come on board an existing project where no one knows how it works?

The Takeaway?

  • Design and testing aren't what slows down a project
  • Software projects never die!
