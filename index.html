HTTP Basics

Creating an API over the Web

  • Verbs
  • Response Codes
  • ETAGs, caching, flat vs. nested, etc. etc.

Verbs

They map to database operations

  • GET = select
  • POST = insert
  • PUT = update
  • DELETE = delete

Use the right tool for the job!

Why do we care?

There's a right (ish) way to do it

  • Intuitive APIs
  • Cleaner URLs
  • Leverage existing code (dojo/ext stores, backbone fetch)

Exceptions

  • URLs have a maximum length (POST for spatial queries)
  • DELETE is often disabled at the server level

Response Codes

  • 200 OK!
  • 4XX The client screwed something up
  • 5XX The server screwed something up
