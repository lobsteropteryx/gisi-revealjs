SOLID

WTH is SOLID?

  • A set of design principles
  • Primarily apply to OO classes (from ~2001)
  • Michael Feathers, Robert Martin

Single Responsibility

A class should...

  • Have a single reason to change
  • Ex: MVC, MVVP

Open/Closed

A class should...

  • Be open for extension, closed for modification
  • Ex: Glimpse data, SpinalMap

Liskov Substitution

A class should...

  • Be substitutable for its base type
  • Ex: Geometry vs. Point, Polygon, etc.

Interface Segregation

A class should...

  • Have different interfaces for different clients
  • Ex: ArcObjects (PointClass implements 30 interfaces)

Dependency Inversion

A class should...

  • Depend on abstractions, not implementation
  • Ex: IDbContext, ArcObjects - NearestNeighbor
